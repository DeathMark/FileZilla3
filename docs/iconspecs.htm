FileZilla 3 Icon set specification

Introduction:

This document describes the requirements for icon sets in FileZilla 3.

Please remember that if you want to create an icon set for FileZilla, that you should be available to update the icon set in the feature when it's required. Else it could happen that the icon set will be removed.

Requirements:

Since FileZilla is Free Software, the same has to apply to all artwork used in FileZilla. This means that all artwork has to be published under the GPL or a GPL compatible license. Note that Creative Commons is not compatible with the GPL.

To improve accessibility of the software, the icons should be available in different sizes. Despite a few exception (see below), there are no restrictions regarding the icon size.

The most commonly sizes used for icons are:

  • 16x16 (the default icon size used by FileZilla)
  • 32x32
  • 48x48

The following icons have to be available in 16x16 (see further below what the icons should mean):

  • file.png
  • folder.png
  • folderclosed.png
  • download.png
  • downloadadd.png
  • upload.png
  • uploadadd.png
  • server.png

The following icons have to be available in 32x32:

  • file.png
  • folder.png
  • folderclosed.png

It is important, that the icons are good-looking in all those sizes.
As fileformat for the icons, the png format has to be used. For each provided size there has to be a subdirectory with the size as name ("16x16" for example) containing all icons in the corresponding size.

If possible, despite the files for the different sizes, each icon should be available in a vector based format (e.g. SVG), so that differently sized icons can quickly be generated.

Regarding the used color-depth there is no limitation.

Each icon set should follow a theme, the icons should match nicely. Also, the function of the icon has to be easily recognizeable.

Required icons:

The following icons together with the required file name are required in all icon sets for FileZilla:

General toolbar:

  • sitemanager.png - Sitemanager
  • localtreeview.png, remotetreeview.png, logview.png, queueview.png - Togglebuttons for the local folder tree, for the remote folder tree, the message log, and the transfer queue.
  • refresh.png - a refresh button
  • processqueue.png - a button that starts/stops transfer queue processing
  • cancel.png, disconnect.png, reconnect.png - cancel, disconnect and reconnect buttons
  • help.png - a help button

Browsing toolbar:

  • synchronize.png - Synchronize folder views (if you change the local dir, remote dir will be changes as well)
  • compare.png - Compare local and remote directories (file entries in the file lists will receive a coloured background if they exist either only on the local or remote side, or when the file attributes differ.)
  • showhidden.png - show hidden files
  • filter.png - filter filenames

Transfer settings toolbar:

  • ascii.png, binary.png, auto.png - Transfermode buttons for Ascii, binary and Auto mode.
  • speedlimits.png - Button to enable speed limits

Context menus

  • upload.png - Upload selected local files
  • uploadadd.png - Add selected local files to queue
  • download.png - Download selected remote files
  • downloadadd.png - Add selected remote files to queue

Other generic icons:

  • bookmars.png - Bookmarks/Favourites
  • folderup.png - "Go up one folder"
  • folderback.png - "Go back to previous folder"
  • file.png - icon for generic files
  • folder.png - icon for open folder/directory
  • folderclosed.png - icon for a closed folder/directory
  • server.png - server icon for the site entries in the Site Manager, can be 16x16 only
